GPIO寄存器级控制（STM32F103 PA5 LED）

核心寄存器

RCC_APB2ENR (时钟使能寄存器)
0x00000000 (未使能GPIOA)
GPIOA_CRL (模式寄存器)
0x44444444 (默认输入模式)
GPIOA_BSRR (置位/复位寄存器)
0x00000000

LED状态

核心代码示例

#define PERIPH_BASE 0x40000000UL
#define GPIOA_BASE (PERIPH_BASE + 0x10800UL)
#define GPIOA_CRL *(volatile uint32_t *)(GPIOA_BASE + 0x00UL)
#define GPIOA_BSRR *(volatile uint32_t *)(GPIOA_BASE + 0x10UL)
#define RCC_APB2ENR *(volatile uint32_t *)(0x40021000UL + 0x18UL)

// 初始化PA5为输出模式
void LED_Init(void) {
  RCC_APB2ENR |= (1UL << 2); // 使能GPIOA时钟
  GPIOA_CRL &= ~(0xFUL << 20); // 清除PA5配置
  GPIOA_CRL |= (0x3UL << 20); // PA5推挽输出
}

void LED_On(void) {
  GPIOA_BSRR = (1UL << 5); // 置位PA5
}

void LED_Off(void) {
  GPIOA_BSRR = (1UL << 21); // 复位PA5
}

关键知识点

  • volatile关键字：确保编译器不优化寄存器读写操作
  • BSRR寄存器：原子操作，避免中断导致的操作不完整
  • 时钟使能：外设默认关闭时钟，必须先使能才能操作
  • 模式配置：每个引脚占用4位，配置输入/输出模式

位带操作（Bit-Banding）原理与演示

位带地址映射

外设位带区：0x40000000 ~ 0x400FFFFF → 别名区：0x42000000 ~ 0x43FFFFFF

计算公式：别名地址 = 0x42000000 + (目标地址 - 0x40000000)×32 + 位号×4

PA5位带别名地址计算
GPIOA_ODR地址：0x4001080C
偏移量：0x4001080C - 0x40000000 = 0x1080C
别名地址：0x42000000 + 0x1080C×32 + 5×4 = 0x42210194

位带操作代码对比

// 传统位运算方式
GPIOA_ODR ^= (1UL << 5); // 需要位运算，非原子操作

// 位带操作方式
#define BITBAND_PERIPH(addr, bit) \   ((volatile uint32_t *)(0x42000000 + ((addr - 0x40000000) << 5) + (bit << 2)))

#define PA5_OUT (*BITBAND_PERIPH(GPIOA_ODR_ADDR, 5))
PA5_OUT = !PA5_OUT; // 直接操作单个位，简洁且原子操作

位带操作优势

  • 代码简洁：无需复杂的位运算（&/|/~）
  • 原子操作：硬件保证单个位的读写不会被中断打断
  • 直观易懂：直接映射到单个引脚，符合直觉
  • 通用性强：适用于所有Cortex-M系列芯片

启动流程时间线

1. 上电复位

CPU复位，硬件自动加载向量表第一个值到栈指针(SP)

2. 加载复位中断地址

从向量表第二个位置读取Reset_Handler地址并执行

DCD __initial_sp ; 栈顶地址
DCD Reset_Handler ; 复位中断服务函数
DCD NMI_Handler ; NMI中断
DCD HardFault_Handler ; 硬故障中断

3. 初始化.data段

将Flash中存储的初始化数据复制到RAM中的.data段

4. 初始化.bss段

将RAM中的.bss段全部清零（未初始化全局变量）

5. 调用系统初始化

执行__libc_init_array，初始化C库环境

6. 跳转到main()

执行用户代码，进入main函数的无限循环

各段初始化代码（汇编）

Reset_Handler:
  ; 1. 设置栈指针
  LDR sp, =__initial_sp

  ; 2. 初始化.data段（Flash → RAM）
  LDR r0, =__data_start__
  LDR r1, =__data_end__
  LDR r2, =__data_load__
  movs r3, #0
  b LoopCopyDataInit
CopyDataInit:
  ldr r4, [r2, r3]
  str r4, [r0, r3]
  adds r3, r3, #4
LoopCopyDataInit:
  cmp r0, r1
  bcc CopyDataInit

  ; 3. 初始化.bss段（清零）
  LDR r0, =__bss_start__
  LDR r1, =__bss_end__
  movs r2, #0
  b LoopFillZerobss
FillZerobss:
  str r2, [r0]
  adds r0, r0, #4
LoopFillZerobss:
  cmp r0, r1
  bcc FillZerobss

  ; 4. 跳转到main()
  bl main
  B . ; main返回后死循环

核心组件说明

  • 向量表（Vector Table）：存储中断服务函数地址，复位后首先读取
  • 启动文件（startup.s）：汇编编写，负责底层初始化，连接硬件和C语言
  • 分散加载文件（.sct）：定义内存布局，告诉链接器如何分配Flash/RAM
  • .text/.data/.bss：分别存储代码、初始化变量、未初始化变量
